Abstract

Choosing a stop (pickup/drop-off or delivery) is a systems problem, not a lookup: the stop must be on a drivable lane, legal to stop at, walkable to the door, and globally coordinated so demand spikes do not converge on one curb. Pin-Service is a tested reference implementation of that serving path, candidate generation, constraint filtering, ML scoring, congestion-aware re-ranking, and load shedding, behind a gRPC API.

Honest scope. This is a clean, tested reference implementation of the canonical serving path (unit tests for each stage + a Locust load test). The engineering pattern is real and runnable; the ML scorer is a gradient-boosted regressor trained on a synthetic satisfaction label and the map / supply signals are fixtures, so the model and data are illustrative.

Validated output

A tested reference serving implementation (~1,100 LOC; 20 unit tests per stage + a Locust load test). The serving pattern is reproduced and runnable; the scorer model and map/supply data are illustrative. Reproduced 2026-06-29 (Locust, 200 users, 60 s, gRPC localhost): served curb-selection latency p50 1 ms / p95 2 ms / p99 6 ms; under a single-curb hotspot flood (~1,495 req/s offered) the congestion fail-safe shed ~99.8% of requests at ~0.26 ms fast-reject with 0 server errors, holding served tail latency in single-digit ms.
